Pebble automaton

In computer science, a pebble automaton is an extension of tree walking automata which allows the automaton to use a finite amount of "pebbles", used for marking tree node. The result is a model stronger than ordinary tree walking automata, but still strictly weaker than branching automata.

Definitions

A pebble automaton is a tree walking automaton with an additional finite set of fixed size containing pebbles, identified with { 1 , 2 , … , n } 1 2 normal-… n \{1,2,\dots,n\} . Besides ordinary actions, an automaton can put a pebble at a currently visited node, lift a pebble from the currently visited node and perform a test "is the i-th pebble present at the current node?". There is an important stack restriction on the order in which pebbles can be put or lifted - the i+1-th pebble can be put only if the pebbles from 1st to i-th are already on the tree, and the i+1-th pebble can be lifted only if pebbles from i+2-th to n-th are not on the tree. Without this restriction, the automaton has undecidable emptiness and expressive power beyond regular tree languages.

Properties

  • P A ⊊ R E G P A R E G PA\subsetneq REG , i.e. pebble automata are strictly weaker than branching automata
  • it is not known whether D P A = P A D P A P A DPA=PA , i.e. whether pebble automata can be determinized

Automata and logic

Pebble automata admit an interesting logical characterization. Let F O + T C F O T C FO+TC denote the set of tree properties describable in transitive closure first-order logic, and F O + pos T C F O pos T C FO+\text{pos}\,TC the same for positive transitive closure logic, i.e. a logic where the transitive closure operator is not used under the scope of negation. Then it can be proved that P A ⊆ F O + T C P A F O T C PA\subseteq FO+TC and, in fact, P A = F O + pos T C P A F O pos T C PA=FO+\text{pos}\,TC - the languages recognized by pebble automata are exactly those expressible in positive transitive closure logic.
