Utility functions on indivisible goods

Some branches of economics and game theory deal with indivisible goods – discrete items that can be traded only as a whole. For example, in combinatorial auctions there is a finite set of items, and every agent can buy a subset of the items, but an item cannot be divided between two or more agents.

It is usually assumed that every agent assigns subjective utility to every subset of the items. This can be represented by one of two ways:

  • An ordinal utility preference relation, usually marked by ≻ succeeds \succ . The fact that an agent prefers a set A A A to a set B B B is written A ≻ B succeeds A B A\succ B . If the agent only weakly prefers A A A (i.e. either prefers A A A or is indifferent between A A A and B B B ) then this is written A ⪰ B succeeds-or-equals A B A\succeq B .
  • A cardinal utility function, usually marked by u u u . The utility an agent gets from a set A A A is written u ( A ) u A u(A) . Cardinal utility functions are often normalized such that u ( ∅ ) = 0 u 0 u(\emptyset)=0 , where ∅ \emptyset is the empty set.

A cardinal utility function implies a preference relation u ( A ) > u ( B ) u A u B u(A)>u(B) implies A ≻ B succeeds A B A\succ B and u ( A ) ≥ u ( B ) u A u B u(A)\geq u(B) implies A ⪰ B succeeds-or-equals A B A\succeq B .

Utility functions can have several properties.1

Monotonicity

Monotonicity means that an agent always (weakly) prefers to have extra items. Formally:

  • For a preference relation A ⊇ B B A A\supseteq B implies A ⪰ B succeeds-or-equals A B A\succeq B .
  • For a utility function A ⊇ B B A A\supseteq B implies u ( A ) ≥ B u A B u(A)\geq B (i.e. u is a monotone function).

Monotonicity is equivalent to the free disposal assumption: if an agent may always discard unwanted items, then extra items can never decrease the utility.

Additivity (also called: linearity) means that "the whole is equal to the sum of its parts". I.e, the utility of a set of items is the sum of the utilities of each item separately. This property is relevant only for cardinal utility functions. It says that for every set A A A :

An additive utility function is characteristic of independent goods. For example, an apple and a hat are considered independent: the utility a person receives from having an apple is the same whether or not he has a hat, and vice versa. Submodularity means that "the whole is not more than the sum of its parts (but may be less)". Formally, for all sets A A A and B B B :

u ( A ) + u ( B ) ≥ u ( A ∪ B ) + u ( A ∩ B ) u A u B u A B u A B u(A)+u(B)\geq u(A\cup B)+u(A\cap B) In other words, u u u is a submodular set function.

An equivalent property is Diminishing marginal utility, which means that for every sets A A A and B B B with A ⊆ B A B A\subseteq B , and every x ∉ B x B x\notin B :2

u ( A ∪ { x } ) - u ( A ) ≥ u ( B ∪ { x } ) - u ( B ) u A x u A u B x u B u(A\cup\{x\})-u(A)\geq u(B\cup\{x\})-u(B) .

A submodular utility function is characteristic of substitute goods. For example, an apple and a bread loaf can be considered substitutes: the utility a person receives from eating an apple is smaller if he has already ate bread (and vice versa), since he is less hungry in that case. Supermodularity is the opposite of submodularity: it means that "the whole is not less than the sum of its parts (but may be more)". I.e, if A A A and B B B are sets, then:

u ( A ) + u ( B ) ≤ u ( A ∪ B ) + u ( A ∩ B ) u A u B u A B u A B u(A)+u(B)\leq u(A\cup B)+u(A\cap B) In other words, u u u is a supermodular set function.

An equivalent property is Increasing marginal utility, which means that for all sets A A A and B B B with A ⊆ B A B A\subseteq B , and every x ∉ B x B x\notin B :

u ( A ∪ { x } ) - u ( A ) ≤ u ( B ∪ { x } ) - u ( B ) u A x u A u B x u B u(A\cup\{x\})-u(A)\leq u(B\cup\{x\})-u(B) .

A supermoduler utility function is characteristic of complementary goods. For example, an apple and a knife can be considered complementary: the utility a person receives from an apple is larger if he already has a knife (and vice versa), since it is easier to eat an apple after cutting it with a knife. A possible utility function for this case is given at the right.

A utility function is additive if and only if it is both supermodular and submodular.

Special types of submodular utilities

Because of their relation to diminishing marginal utility, submodular utility functions are very common in economics. Several sub-families of the submodular family are described below, in order of containment, from the more specific to the more general.
